# 20260331-slice_3a.md
## Slice 3A: Non-interactive Session Continuity

**Date:** 2026-03-31
**Status:** Approved for implementation
**Depends on:** Slice 0 (Verified), Slice 1 (Verified), Slice 2 (Verified)

**Slice 3A goal:** Replace the original Slice 3 execution path with a safer primary continuity path.
Continuity is proved without live multi-turn spoken interaction as the main acceptance vehicle.
Session lifecycle, turn artifact persistence, bounded working memory, and explicit memory write policy are implemented and validated through deterministic, non-interactive execution.

The completed loop after Slice 3A:
```
hardware-aware startup
→ session opened, session ID assigned
→ turn 1 executed from explicit transcript input
→ response produced through the normal cognition path
→ turn 1 artifact written
→ working memory updated by explicit policy
→ turn 2 executed from explicit transcript input
→ prior turn context injected explicitly into prompt assembly
→ turn 2 artifact written
→ session ends cleanly
```

This slice preserves the original product direction from `ProjectVision.md`:
continuity remains explicit, artifacts remain authoritative, and text/non-interactive ingress is still routed through the same turn engine rather than a separate product architecture.

This slice is intentionally **not** the live spoken multi-turn proof.
That validation moves to Slice 3B after the continuity path is already proved.

No SQLite. No vector store. No semantic retrieval. No summarization.
Filesystem artifacts + bounded in-process working memory only.

---

## Current Repo State and Why This Split Exists

The governed repo state currently proves:

- Slice 0 hardware profiler is verified
- Slice 1 minimal voice turn is verified
- Slice 2 spoken response via local Kokoro TTS is verified

The current inventory and change log also show that the verified runtime is strongest at the single-turn boundary.
They do **not** yet prove multi-turn spoken continuity.

The original Slice 3 combined two concerns too early:

1. continuity architecture
2. fragile live spoken multi-turn validation

That coupling is unnecessary risk.
Slice 3A therefore becomes the authoritative continuity implementation slice.
Slice 3B becomes a thin live spoken validation layer that reuses the 3A path rather than defining it.

---

## Runtime Prerequisites

All Slice 2 prerequisites carry forward where applicable. For Slice 3A primary acceptance, the required gates are narrower:

1. **`data/sessions/` writable** — session state files are written here
2. **`data/turns/` writable** — turn artifacts are written here
3. **Ollama reachable at the existing Slice 1/2 runtime boundary** — non-interactive runtime validation still uses the current LLM path
4. **Microphone not required for primary acceptance** — Slice 3A does not depend on live capture
5. **Audio output device not required for primary acceptance** — Slice 3A does not depend on live spoken playback

If microphone or output devices are unavailable, Slice 3A can still be accepted.
If the non-interactive runtime path cannot execute through the current LLM boundary, Slice 3A is not complete.

## Sub-Slice 3A.1 — Session Lifecycle Manager

### Objective
Introduce explicit session identity and disk-backed lifecycle state without touching the live spoken runtime yet.
The session is the continuity boundary for turn grouping, artifact storage, and memory policy.

### Files to Create

**`backend/app/conversation/session_manager.py`**

```python
@dataclass
class Session:
    session_id: str
    started_at: str
    ended_at: str | None
    turn_count: int

class SessionManager:
    def open_session() -> Session: ...
    def increment_turn(session: Session) -> Session: ...
    def close_session(session: Session) -> Session: ...
```

Session state file path:
`data/sessions/<session_id>.json`

Session file schema:
```json
{
  "session_id": "...",
  "started_at": "...",
  "ended_at": null,
  "turn_count": 0
}
```

Implementation rules:

- `open_session()` creates `data/sessions/` if absent
- all writes use atomic write (`.tmp` then rename)
- `increment_turn()` updates the session file immediately after a completed persisted turn
- `close_session()` records `ended_at` and writes final session state
- no class-level mutable state; the `Session` dataclass + disk file are the only sources of truth

This is the explicit continuity evidence layer required by `ProjectVision.md`.

```python

    turn_id: str
    session_id: str
    turn_index: int

    transcript: str
    prompt_text: str
    response_text: str
    personality_profile_id: str
    stt_model: str | None
```

Artifact file path:
`data/turns/<session_id>/<turn_id>.json`

Implementation rules:

- directories created automatically when absent
- JSON serialization only; no new storage dependency

## Sub-Slice 3A.3 — Working Memory and Explicit Prompt Context Injection

### Objective
Add bounded in-process working memory and extend prompt assembly so prior session turns are injected explicitly.
This is the continuity mechanism for Slice 3A.

### Files to Create / Modify

**`backend/app/memory/working.py`**

```python
@dataclass
class TurnSummary:
    turn_index: int
    transcript: str
    response_text: str

class WorkingMemory:
    def __init__(self, max_turns: int = 5): ...
    def add_turn(self, summary: TurnSummary) -> None: ...
    def get_context_turns(self) -> list[TurnSummary]: ...
    def clear(self) -> None: ...
```

Rules:

- bounded rolling window only
- oldest turn dropped when `max_turns` exceeded
- no file I/O in working memory
- no reconstruction-on-restart in Slice 3A

**`backend/app/cognition/prompt_assembler.py`** — extend current function:

```python
def assemble_prompt(
    transcript: str,
    personality: PersonalityProfile,
    context_turns: list[TurnSummary] | None = None,
) -> str:
```

When `context_turns` is non-empty, prepend explicit structured prior turns before the current user turn.
Required format shape:

```
[System: <identity_summary>. Tone: <tone>.]

[Prior turns:]
User: <turn_0.transcript>
Assistant: <turn_0.response_text>
...

User: <current transcript>
```

When `context_turns` is `None` or empty, behavior remains compatible with Slice 1/2.

## Sub-Slice 3A.4 — Explicit Working-Memory Write Policy

### Objective
Make memory write behavior explicit, bounded, and inspectable.
No implicit writes.
No hidden memory side effects.

### Files to Create

**`backend/app/memory/write_policy.py`**

```python
@dataclass
class WriteDecision:
    should_write: bool
    reason: str

def evaluate_write_policy(
    transcript: str,
    response_text: str,
    final_state: str,
) -> WriteDecision:
```

Policy rules for Slice 3A:

| Condition | Decision | Reason |
|-----------|----------|--------|
| `final_state == "FAILED"` | skip | `"turn failed — not written to working memory"` |
| `transcript.strip() == ""` | skip | `"empty transcript — not written to working memory"` |
| `response_text.strip() == ""` | skip | `"empty response — not written to working memory"` |
| otherwise | write | `"turn complete — written to working memory"` |

Rules:

- pure function only
- no I/O
- the artifact is still written even when working memory skips the turn
- write policy governs only working memory admission

## Sub-Slice 3A.5 — Canonical Transcript-Bound Turn Executor

### Objective
Create the continuity execution surface used by Slice 3A primary validation.
This executor starts at the transcript boundary and drives the same cognition/runtime stack already proved in Slices 1 and 2, but without live microphone or spoken playback dependence.

### Files to Create

**`backend/app/services/turn_service.py`**

```python
def run_turn(
    report: FullCapabilityReport,
    personality: PersonalityProfile,
    transcript: str,
    *,
    session: Session | None = None,
    memory: WorkingMemory | None = None,
    input_modality: str = "text",
) -> str:
    ...
```

Required sequence:

```
1. engine = ConversationEngine(report, personality)
2. engine.transition(REASONING)
3. llm = select_llm_runtime()
4. prompt = assemble_prompt(transcript, personality, context_turns=memory.get_context_turns() if memory else None)
5. engine.transition(RESPONDING)
6. response = get_response(prompt, llm)
7. engine.transition(IDLE)
8. decision = evaluate_write_policy(transcript, response, engine.state.name)
9. print(f"[MEMORY] {decision.reason}")
10. if decision.should_write and memory is not None: memory.add_turn(...)
11. if session is not None: build TurnArtifact, write_turn_artifact(...), SessionManager.increment_turn(session)
12. return response
```

Rules:

- no microphone capture in this function
- no TTS/playback in this function
- artifact `stt_model` is `None` for text/non-interactive ingress
- artifact `tts_runtime` is `None` in Slice 3A
- when `session` or `memory` is `None`, the function behaves as a stateless single-turn executor
- any exception transitions to `FAILED`, records explicit failure reason where possible, and re-raises

This is the authoritative continuity executor for Slice 3A.
The live spoken path is not yet the authority.

## Key Design Decisions

**Slice 3A is the authoritative continuity path.**
Continuity is first proved at the transcript boundary inward, where the repo is already strongest and most deterministic.

**The artifact is always authoritative.**
Whether a turn is admitted to working memory is a policy decision.
Whether a turn is recorded as an artifact is not optional once the turn completes.

**Working memory is bounded and in-process only.**
No restart reconstruction, no semantic indexing, no hidden long-term memory behavior.
That keeps this slice aligned with current repo scope and explicit-cognition governance.

**Prompt-context proof is stronger than answer-wording proof for this slice.**
Slice 3A proves continuity by showing that prior-turn context is deterministically injected into the second prompt and persisted in artifacts.
That is more reliable than requiring a specific LLM phrasing.

**No live spoken multi-turn acceptance in Slice 3A.**
That is deferred intentionally to Slice 3B, where it becomes a thin runtime validation layer over an already-proved continuity path.
