## Slice 4 Goal

Make barge-in a first-class, traceable system behavior.

When JARVIS is speaking, the user can interrupt. Speech stops. The interruption is recorded in the turn artifact. The system transitions cleanly to a state where the user can speak again. No session corruption. No silent failure. No hang.

The completed interaction loop after Slice 4:

```
hardware-aware startup
→ session opened
→ turn 1: listen → transcribe → reason → respond → speak
→ user speaks during SPEAKING phase → barge-in accepted
→ speech stops at nearest safe boundary
→ INTERRUPTED state recorded with artifact
→ system transitions cleanly: INTERRUPTED → RECOVERING → LISTENING
→ turn 2: second speech captured → transcribed → cognition with context → response → speak
→ session closes cleanly
```

`INTERRUPTED` and `RECOVERING` already exist in `ConversationState`. Slice 4 gives them their first real implementation.

---

## Architectural Approach

### The core problem

`play_audio()` in `playback.py` currently calls `sd.play()` + `sd.wait()` — a blocking call. There is no hook to stop it externally. Simultaneously listening for a barge-in signal on the microphone while playing audio requires both to operate concurrently. That means threading.

### The chosen approach: threading-based concurrent listen + play

The barge-in listener runs on a background thread while TTS plays on the main voice path. A shared `threading.Event` — called the **interrupt flag** — is the sole communication channel between the listener and the playback path. When the listener detects a valid barge-in signal, it sets the flag. The playback wrapper polls the flag and calls `sd.stop()` at the nearest safe point.

This is the minimal threading surface needed. It does not introduce async, coroutines, multiprocessing, or a new event bus. It extends the existing `sounddevice` usage pattern already in the repo.

### Why not VAD-based barge-in yet

VAD (voice activity detection) is the production-quality approach. It detects real speech energy rather than responding to any audio above a threshold. However VAD introduces a new dependency (`webrtcvad` or `silero-vad`) and a new model/config surface. For Slice 4, a threshold-based energy detector on the input device is sufficient to prove the barge-in architecture. VAD can be introduced in a later slice as a drop-in replacement for the detection function.

### Architectural risk: `sounddevice` simultaneous input + output

`sounddevice` (PortAudio) supports simultaneous input and output streams on most modern Windows hosts but requires them to be opened as separate streams — the default stream opened by `sd.play()` is output-only. The barge-in listener must open a separate input stream via `sd.InputStream`. This is supported by `sounddevice` but has not been exercised in the repo yet. The 4.2 sub-slice validates this on the actual host before wiring it into the full live path.

### PortAudio stream sequencing rule (inherited from Slice 3B)

`sd.stop()` is already called before `sd.rec()` in `stt_runtime.py` to avoid stream conflicts. The same discipline applies here: when the barge-in path stops TTS playback, it calls `sd.stop()` to clear the output stream before the recovery path attempts to open a new input stream for the next turn.

## Sub-Slice 4.2 — Barge-In Signal Detector

### Objective

Implement a standalone, independently testable barge-in detection utility that can run on a background thread while audio is playing. Validate that simultaneous input/output streams work on this host before wiring into the voice path.

### Files to Create

**`backend/app/runtimes/stt/barge_in.py`**

```python
import threading

class BargeInDetector:
    """Listens on the default input device for a barge-in signal.

    Runs on a background thread. When energy above threshold is detected
    for at least min_frames consecutive frames, sets the interrupt_flag.

    Design rules:
    - Separate sd.InputStream from the TTS output stream — no shared stream.
    - Calls sd.stop() on exit to release the input stream cleanly.
    - Never raises from the background thread; logs and sets a failure flag.
    - threshold and min_frames are tunable without code changes.
    """

    def __init__(
        self,
        interrupt_flag: threading.Event,
        *,
        threshold: float = 0.02,          # RMS energy threshold; tunable
        min_trigger_frames: int = 3,       # consecutive frames above threshold before triggering
        sample_rate: int = 16000,
        block_size: int = 1024,
    ) -> None: ...

    def start(self) -> None:
        """Open input stream and start background detection thread."""

    def stop(self) -> None:
        """Stop detection thread and close input stream cleanly."""

    @property
    def failed(self) -> bool:
        """True if the detector encountered an unrecoverable error."""

    @property
    def failure_reason(self) -> str | None:
        """Human-readable failure reason if failed is True, else None."""
```

Implementation notes:

- Uses `sd.InputStream` with a callback — this does not conflict with a concurrent `sd.play()` output stream.
- The callback computes per-block RMS: `rms = float(np.sqrt(np.mean(indata**2)))`.
- A consecutive-frames counter increments when `rms > threshold`, resets when below. When the counter reaches `min_trigger_frames`, `interrupt_flag.set()` is called.
- `stop()` calls `stream.stop()` and `stream.close()` — it does NOT call `sd.stop()` (which would terminate the concurrent output stream). Only the voice service orchestration layer calls `sd.stop()` to clear the output after playback is interrupted.
- If the input stream cannot be opened (no input device, PortAudio error), `failed` is set to `True` and `failure_reason` is populated. The interrupt flag is never set. The voice path must check `detector.failed` before and after playback and degrade cleanly.

## Sub-Slice 4.3 — Interruptible Playback

### Objective

Replace the current blocking `play_audio()` with an interruption-aware wrapper that monitors the interrupt flag and stops playback when it is set.

### Files to Modify

**`backend/app/runtimes/tts/playback.py`**

Add alongside the existing `play_audio()`:

```python
def play_audio_interruptible(
    audio_path: str,
    interrupt_flag: threading.Event,
    *,
    poll_interval_seconds: float = 0.05,
) -> bool:
    """Play audio from audio_path. Returns True if playback completed normally,
    False if interrupted (interrupt_flag was set before completion).

    Polls interrupt_flag every poll_interval_seconds. On interruption:
    - calls sd.stop() to halt the output stream
    - returns False immediately

    Raises RuntimeError if the file is not found or no output device is available.
    Does not raise on interruption — returns False instead.
    """
```

Implementation notes:

- Opens with `sf.read()` then `sd.play()`, same as current `play_audio()`.
- Instead of `sd.wait()`, polls `interrupt_flag.is_set()` in a loop using `sd.get_stream().active` to detect natural completion.
- On natural completion: calls `sd.stop()` (existing pattern), returns `True`.
- On interruption detected: calls `sd.stop()`, returns `False`.
- The existing `play_audio()` is unchanged — the non-interruptible path remains available for callers that do not need barge-in (e.g. non-interactive tests).

## Sub-Slice 4.4 — Interruptible Voice Turn Orchestration

### Objective

Wire `BargeInDetector` and `play_audio_interruptible()` into `run_voice_turn()`. Handle the `INTERRUPTED` → `RECOVERING` → `LISTENING` transition path. Write the interruption artifact.

### Files to Modify

**`backend/app/services/voice_service.py`**

Extend the `SPEAKING` section of `run_voice_turn()`:

```
engine.transition(SPEAKING)
tts = select_tts_runtime(report)
if tts is None:
    print("[DEGRADED] TTS unavailable — text response only")
    engine.transition(IDLE)
    return response

tts_audio_path = str(Path(temp_dir) / "response.wav")
tts.synthesize(response, tts_audio_path)

# Barge-in path
interrupt_flag = threading.Event()
detector = BargeInDetector(interrupt_flag)
detector.start()

spoken_normally = play_audio_interruptible(tts_audio_path, interrupt_flag)

detector.stop()

if not spoken_normally:
    # Barge-in accepted
    print("[INTERRUPTED] barge-in detected — stopping speech")
    engine.transition(INTERRUPTED)
    # Write artifact with interruption evidence
    # (session/memory write handled via existing run_turn path; artifact update is additive)
    _record_interruption(session=session, turn_index=..., interrupted_at=_now_iso())
    engine.transition(RECOVERING)
    print("[RECOVERING] ready for next turn")
    engine.transition(LISTENING)
    return ""   # empty response signals to caller that interruption occurred; caller decides next action

engine.transition(IDLE)
return response
```

Additional helper:

**`_record_interruption(session, turn_index, interrupted_at)`** — reads the most recently written artifact for this session/turn, reconstructs it with `interrupted=True` and `interrupted_at` set, and rewrites it atomically. Uses the existing `write_turn_artifact()` and `read_turn_artifact()` from `storage.py`.

If `detector.failed` before playback starts: log `[DEGRADED] barge-in detector unavailable — ({detector.failure_reason})`, skip detector entirely, fall back to blocking `play_audio()`. This is the degraded path when simultaneous streams are not available on the host.

### Session/Memory Interaction

When interruption occurs, the response text was already committed to working memory and the artifact was already written by `run_turn()` (the cognition path runs before SPEAKING). The interruption only affects the playback phase. Memory state is therefore consistent — the interrupted turn exists in memory as a completed cognition turn, and the artifact is retroactively updated to record the interruption evidence.

This avoids the complexity of trying to partially undo a turn from memory. The turn happened; JARVIS was interrupted while speaking it. That is the accurate record.

## Key Design Decisions

**`BargeInDetector` owns the input stream; `play_audio_interruptible` owns the output stream.**
They are separate PortAudio streams. One being active does not block the other. The only shared state is the `threading.Event` interrupt flag.

**Interruption returns `""` from `run_voice_turn`.**
The caller (test or future loop) can distinguish interruption from a normal response by checking for empty string. This is a simple, inspectable contract. A future slice can introduce a typed return (e.g. `VoiceTurnResult`) if richer signaling is needed.

**The cognition turn artifact is written before the interruption can occur.**
`run_turn()` completes and writes the artifact before `SPEAKING` begins. The interruption retroactively updates that artifact's `interrupted` fields. This avoids any need to partially undo memory or artifact state on interruption.

**Degraded path is explicit.**
If `BargeInDetector` cannot open an input stream, `detector.failed` is `True`, the fallback is the existing blocking `play_audio()`, and the log line says exactly why. The voice turn completes normally without barge-in capability. No silent fallback.

**`INTERRUPTED` → `RECOVERING` → `LISTENING` is the only legal recovery path.**
Guarded in `ConversationEngine`. Any attempt to transition directly from `INTERRUPTED` to `IDLE` or any other state raises `RuntimeError`. This prevents silent session corruption from an incorrect recovery implementation.

**VAD is deferred.**
The threshold-based energy detector is sufficient to prove the barge-in architecture. It can be replaced with a `webrtcvad` or `silero-vad` call inside `BargeInDetector._callback()` in a future slice without changing the surrounding architecture.
