## Slice 5B Goal

Layer assistant presence, personality shaping, and wake word onto the application shell contract established by Slice 5A.

Slice 5A established the start-path contract and proved it through a bounded local shell adapter. Slice 5B makes that shell feel like JARVIS — responsive, characterful, and hands-free — without redefining what the host is.

Slice 5B adds:
- wake word detection so the user does not need to type `v` to invoke voice
- richer personality shaping controls beyond the current minimal YAML fine-tuning
- pre-turn acknowledgment so the user gets immediate feedback that JARVIS heard them
- voice selection override so persona tuning does not require code changes

**Scope note:** wake word, acknowledgment, and persona shaping are all implemented against the service contract from 5A. They are not coupled to `scripts/run_jarvis.py` specifically — a future desktop shell or richer host can consume the same behavior. The bounded shell adapter is extended in this slice only because it is the current exercising host.

**What Slice 5B is not:** a graphical UI, system tray, overlay window, or hotkey daemon. Those belong after presence/persona is proven in the current shell host.

## Architectural Approach

### Wake word: greenfield implementation via `backend/app/runtimes/wake/`

`repo_tree.md` declares `backend/app/runtimes/wake/` with `base.py`, `local_runtime.py`, and `wakeword_runtime.py`, but these files do not exist yet. Slice 5B is their first implementation.

The chosen engine is **Picovoice Porcupine** (`pvporcupine`):
- local-first, no cloud calls
- free tier available; requires a free Picovoice Access Key (`PICOVOICE_ACCESS_KEY`)
- provides a built-in `"jarvis"` keyword on supported platforms (availability varies by version — degradation to push-to-talk if absent)
- integrates with `sounddevice.InputStream` at 16kHz/512-frame PCM16, the same stream pattern proven in Slice 4 by `BargeInDetector`

**This is a greenfield surface.** No prior wake-word integration exists in the repo. Concurrent stream behavior (Porcupine input + barge-in input during SPEAKING) has not yet been validated on the target host. The 5B.1 sub-slice explicitly validates this before wiring into the shell loop.

**If `PICOVOICE_ACCESS_KEY` is absent or the `"jarvis"` keyword is unavailable, wake word degrades cleanly to push-to-talk.** The shell adapter continues working exactly as it did in Slice 5A.

### Acknowledgment: pre-cognition, personality-aware

The acknowledgment plays immediately after transcription and before `run_turn()` begins. This gives the user perceived responsiveness while the LLM processes. It is implemented as a pure function in the personality layer and consumed by `voice_service.py`.

### Host-agnostic design

All new behavior introduced in Slice 5B is wired through the backend service contract, not through `scripts/run_jarvis.py` directly:
- `select_wake_runtime()` is a backend selector that any host can call
- `generate_acknowledgment()` is a pure function any host can use
- personality schema extensions are YAML-configurable and consumed by `run_startup()` already

When a future desktop or frontend host replaces the script, these capabilities transfer without re-implementation.

## Runtime Prerequisites

All Slice 5A prerequisites carry forward. Additionally:

1. **`PICOVOICE_ACCESS_KEY`** — free key from Picovoice console; must be set in `.env` for wake word; absence → graceful push-to-talk fallback. `.env.example` must document this.
2. **`pvporcupine>=3.0`** — additive to `backend/requirements.txt`.
3. **Built-in `"jarvis"` keyword** — available in Porcupine 3.x on Windows x86_64. If not present on the target host, `select_wake_runtime()` returns `None` and wake word is skipped gracefully.
4. **Simultaneous InputStream + InputStream (Porcupine + BargeInDetector)** — not yet validated on this host. Sub-slice 5B.1 explicitly proves this before shell integration.

## Sub-Slice 5B.1 — Wake Word Runtime

### Objective

Implement the `wake/` runtime layer. Validate simultaneous audio streams before wiring into the shell.

### Files to Create

**`backend/app/runtimes/wake/base.py`**

```python
class WakeWordBase(ABC):
    @abstractmethod
    def start(self, wake_flag: threading.Event) -> None:
        """Begin listening. Set wake_flag when keyword detected."""
    @abstractmethod
    def stop(self) -> None: ...
    @abstractmethod
    def is_available(self) -> bool: ...
    @property
    @abstractmethod
    def failed(self) -> bool: ...
    @property
    @abstractmethod
    def failure_reason(self) -> str | None: ...
```

**`backend/app/runtimes/wake/local_runtime.py`**

`PorcupineWakeWord(WakeWordBase)`:
- `is_available()`: `import pvporcupine` succeeds and `access_key` non-empty
- `start(wake_flag)`: opens `pvporcupine.Porcupine(access_key, keywords=["jarvis"])`, opens `sd.InputStream(dtype="int16", blocksize=512, samplerate=16000)` callback feeding frames to Porcupine; sets `wake_flag` on detection
- `stop()`: `stream.stop(); stream.close(); porcupine.delete()` — no `sd.stop()` call (output stream not owned here)
- failure contract: `failed`/`failure_reason` populated on any exception; never raises from background thread

**`backend/app/runtimes/wake/wakeword_runtime.py`**

```python

    """Return PorcupineWakeWord if available and key present; else None."""
```

**`backend/app/core/capabilities.py`** — extend `BackendReadiness`:
```python

```

**`backend/app/hardware/profiler.py`** — extend `_derive_backend_readiness`:

## Sub-Slice 5B.4 — Wake Word Integration into Shell Adapter

### Objective

Wire `PorcupineWakeWord` into `scripts/run_jarvis.py` so the current shell adapter supports hands-free invocation. This is an extension of the local shell adapter; it does not represent the final wake-word integration pattern for a desktop host.

### Files to Modify

**`scripts/run_jarvis.py`** — extend shell loop:

```
If wake_ready=True:
  - start PorcupineWakeWord in background before shell loop begins
  - poll wake_flag alongside stdin input (see threading pattern below)
  - when wake_flag set: automatically trigger voice turn
  - after turn completes: clear flag and restart listener
  - Ctrl+C / 'q': stop listener cleanly before exit

If wake_ready=False (no key, package missing, keyword unavailable):
  - push-to-talk mode only (5A behavior unchanged)
  - startup summary shows: ⚠ WAKE WORD: unavailable (push-to-talk mode)
```

Threading pattern (stdin + wake flag co-polling):
```python
cmd_queue = SimpleQueue()
stdin_thread = Thread(target=_stdin_reader, args=(cmd_queue,), daemon=True)
stdin_thread.start()

while running:
    if wake_flag.is_set():
        wake_flag.clear()
        _run_voice_turn_in_loop()
        continue
    try:
        cmd = cmd_queue.get(timeout=0.1)
        _dispatch(cmd)
    except Empty:
        pass
```

**Future host note:** A desktop host will not use a stdin loop. Wake word integration for a desktop host is a separate concern — the `select_wake_runtime()` selector and `WakeWordBase` contract are designed to be consumed by any host. The threading pattern above is specific to the bounded local shell adapter.

## Key Design Decisions

**Wake word is greenfield — validate concurrency first.** No prior wake word integration exists. Sub-slice 5B.1 explicitly validates Porcupine + BargeInDetector simultaneous streams before any shell wiring. If the host cannot support concurrent streams, wake word is deferred and the slice continues with push-to-talk only.

**Degrade without key or keyword.** `wake_ready=False` → push-to-talk mode → no error, no hang. Documented in `.env.example`.

**Service contract, not script coupling.** `select_wake_runtime()`, `generate_acknowledgment()`, personality schema extensions — all designed for any host. The shell adapter extension in 5B.4 is a proof-of-concept integration, not the final delivery pattern.

**Acknowledgment is pre-cognition.** Plays immediately after transcription, before `run_turn()`. Gives perceived responsiveness while LLM processes. No randomness — deterministic per `profile_id` hash to keep behavior consistent within a persona.

**Schema extension is additive and backward-compatible.** Three new `PersonalityProfile` fields have Python defaults and `setdefault()` in the loader. All existing profiles and all existing tests remain valid without changes.

**`voice_name_override` is the escape hatch.** Set in YAML to bypass the resolver. Power-user persona tuning without code changes.

**No graphical host in this slice.** Tray icon, overlay, hotkey daemon — after presence/persona is proven in the current shell host, not before.
