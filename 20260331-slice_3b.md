# 20260331-slice_3b.md
## Slice 3B: Live Multi-Turn Spoken Continuity

**Date:** 2026-03-31
**Status:** Approved for implementation after Slice 3A verification
**Depends on:** Slice 0 (Verified), Slice 1 (Verified), Slice 2 (Verified), Slice 3A (Verified)

**Slice 3B goal:** Add one live multi-turn spoken continuity proof on top of the already-verified Slice 3A continuity path.
This slice does not redefine session continuity, memory policy, or artifact structure.
It reuses the 3A path and proves that the live spoken ingress/egress can ride on top of it without expanding the memory system.

The completed loop after Slice 3B:
```
hardware-aware startup
→ session opened
→ turn 1 live speech capture → STT transcript
→ transcript handed to the 3A continuity executor
→ response produced and spoken
→ artifact written through the 3A continuity path
→ turn 2 live speech capture → STT transcript
→ transcript handed to the same 3A continuity executor with prior context
→ response produced and spoken
→ second artifact written
→ session closes cleanly
```

Slice 3B is intentionally thin.
If the live path is unstable, this slice fails closed.
It does **not** broaden into new storage layers, new memory families, or alternate fallback architectures.

---

## Current Repo State and Why 3B Is Separate

The governed repo state already verifies:

- one-turn live voice capture and STT (Slice 1)
- one-turn live spoken response with Kokoro TTS (Slice 2)

The change log also records that runtime instability has occurred at the live boundary before, including transient local runtime failures.
That is exactly why Slice 3B must be a follow-on validation layer rather than the primary continuity implementation vehicle.

Slice 3A proves continuity architecture.
Slice 3B proves that the live spoken runtime can traverse that architecture for two turns.

## Sub-Slice 3B.1 — Reuse the 3A Continuity Executor from the Live Voice Path

### Objective
Make the current live voice path call into the already-proved 3A continuity executor instead of defining a separate multi-turn logic path.

### Files to Modify

**`backend/app/services/voice_service.py`**

Extend the current `run_voice_turn(...)` signature to remain backward-compatible while allowing session continuity inputs:

```python
def run_voice_turn(
    report: FullCapabilityReport,
    personality: PersonalityProfile,
    *,
    session: Session | None = None,
    memory: WorkingMemory | None = None,
) -> str:
    ...
```

Required sequence:

```
1. engine = ConversationEngine(report, personality)
2. engine.transition(LISTENING)
3. audio_path = capture_utterance(...)
4. engine.transition(TRANSCRIBING)
5. stt = FasterWhisperSTT(...)
6. transcript = stt.transcribe(audio_path)
7. print(f"[TRANSCRIPT] {transcript}")
8. hand transcript to the Slice 3A continuity executor
9. continuity executor performs prompt assembly, response generation, write policy, memory update, artifact persistence, and session turn increment
10. if TTS is available: speak response and finish cleanly
11. return response
```

Minimal additional architecture rule:

- do not re-implement session lifecycle in `voice_service.py`
- do not duplicate memory write policy in `voice_service.py`
- do not create a parallel artifact format for the live path
- only add the smallest extension needed so the live voice wrapper can reuse 3A

If `run_voice_turn(...)` needs one additional hook into the 3A executor so artifact completion happens after speech playback, that extension is allowed.
No larger redesign is allowed in this slice.

## Sub-Slice 3B.2 — One Live Multi-Turn Spoken Validation Path

### Objective
Prove one real spoken multi-turn session using the 3A continuity path underneath.
This is the acceptance surface for Slice 3B.

### Files to Create

**`backend/tests/runtime/test_slice3b_multiturn_voice_live.py`**

Single live runtime test:

```python
def test_multiturn_voice_session_live() -> None:
    # explicit prerequisite gates
    # microphone
    # output device
    # STT model
    # TTS model
    # Ollama
    # Slice 3A runtime path assumed verified before this slice begins

    report = run_profiler()
    personality = load_personality_profile("default")
    session = SessionManager.open_session()
    memory = WorkingMemory(max_turns=5)

    print(f"[SESSION] opened: {session.session_id}")

    print("[TURN 1] speak now")
    response1 = run_voice_turn(report, personality, session=session, memory=memory)
    print(f"[TURN 1] response: {response1}")
    print(f"[TURN 1] memory_turns: {len(memory.get_context_turns())}")

    print("[TURN 2] ask about the previous turn")
    response2 = run_voice_turn(report, personality, session=session, memory=memory)
    print(f"[TURN 2] response: {response2}")
    print(f"[TURN 2] memory_turns: {len(memory.get_context_turns())}")

    turn_ids = list_session_turns(session.session_id)
    artifact2 = read_turn_artifact(session.session_id, turn_ids[1])

    session = SessionManager.close_session(session)
    print(f"[SESSION] closed: {session.ended_at}")

    assert len(turn_ids) == 2
    assert len(memory.get_context_turns()) == 2
    assert session.turn_count == 2
    assert session.ended_at is not None
    assert isinstance(response1, str) and len(response1.strip()) > 0
    assert isinstance(response2, str) and len(response2.strip()) > 0
    assert artifact2.input_modality == "voice"
    assert artifact2.prompt_text.strip() != ""
```

Live operator guidance for the two turns:

- **Turn 1:** state one short fact that can be referenced directly
- **Turn 2:** ask about that exact fact

Example:

- Turn 1: `My codename is JARVIS.`
- Turn 2: `What codename did I give you?`

Slice 3B runtime acceptance should inspect both:

1. live spoken behavior actually completed twice
2. second-turn artifact prompt contains prior-turn context from the 3A path

This keeps 3B grounded in live runtime evidence while still using explicit continuity proof rather than depending only on subjective answer wording.

## Sub-Slice 3B.3 — Fail-Closed Behavior, Runtime-Only Acceptance, and Governance Gates

### Objective
Keep the live path honest.
If the live spoken route is unstable, fail closed and surface the reason explicitly.

### Required Fail-Closed Rules

1. **No silent fallback to the non-interactive path for acceptance.**
   Slice 3B may reuse 3A internally, but if live capture/STT/TTS fails, the live slice fails.

2. **No automatic retry loop inside the acceptance test.**
   The change log already shows that transient runtime failures can occur.
   Slice 3B must surface first-failure behavior rather than hiding it behind reruns.

3. **No text-only downgrade counted as success.**
   That was acceptable in earlier slices as degraded behavior.
   It is not sufficient for Slice 3B acceptance because the slice objective is live spoken continuity.

4. **Failure reason must be named.**
   Examples:
   - `[PREREQ FAILED] microphone unavailable`
   - `[LIVE PATH FAILED] transcription failed`
   - `[LIVE PATH FAILED] Ollama unavailable during turn 2`
   - `[LIVE PATH FAILED] TTS playback failed`

5. **No new memory/storage expansion is allowed as a workaround.**
   Do not add summarization, semantic retrieval, vector storage, or new persistence families in response to live instability.

### Validation Model

Slice 3B is **runtime-first and runtime-only for acceptance**.
3A already carries the deterministic/unit burden for continuity logic.
3B adds only the live spoken validation burden on top.

## Key Design Decisions

**Slice 3A remains the continuity authority.**
Slice 3B does not redefine continuity logic.
It only proves that the already-established continuity path survives two live spoken turns.

**Live spoken validation is intentionally thin.**
This slice adds the minimum wiring needed to bridge microphone/STT/TTS into the 3A continuity executor.
Nothing more.

**Fail-closed behavior is required.**
If the live runtime is unstable, the slice stays incomplete.
The repo should not claim multi-turn spoken continuity on the strength of a hidden fallback.

**No broader memory expansion belongs here.**
No semantic retrieval, no long-term memory system, no summarization layer, no new storage engine.
Those are outside the scope of this split.

**Artifact evidence still matters in the live slice.**
Even in 3B, the strongest proof of continuity is that the second-turn artifact prompt contains prior-turn context and the session/artifact counts are correct.
Live spoken success sits on top of that evidence, not instead of it.
